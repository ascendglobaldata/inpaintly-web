# Weekly Themes — 16 Week Schedule

Source of truth for Inpaintly's weekly drops. Humans edit this file.
Claude syncs any changes into `src/lib/themes.json` (the file the app
reads). Do **not** edit `themes.json` by hand.

## How the rollover works

- **One theme per week.** Rolls over automatically at **Monday 08:00 America/Los_Angeles**.
- **Four outfits per theme.** They render as the 4 tiles on the picker.
- **One sample image per outfit** lives at `/public/samples/<slug>/<id>.jpg`.
  Generated by `scripts/generate-samples.mjs`.
- **The schedule loops.** Selection is index-based against a rolling counter
  anchored on 2026-04-20 (REFERENCE_MONDAY in `src/lib/activeWeek.ts`).
  With N weeks in the array, week (N+1) shows the same theme as week 1,
  week (N+2) = week 2, and so on forever. Once the library hits ~30 weeks
  the loop is long enough that no repeat user sees the same theme twice
  within half a year.
- **Extending.** To add more themes, append to the end of `weeks` in
  `themes.json` (Claude handles this when you update this doc). Next time
  the counter reaches the new index, it'll appear.
- **Override for testing.** Set `active_week` in `themes.json` to a specific
  `week_of` date. That week renders regardless of the counter. Remove to
  resume normal rotation.
- **Previous week slot.** The theme picker shows two sections stacked:
  current drop on top, previous drop below (both with tagged pills). The
  "previous" slot defaults to `currentIndex − 1` with wrap-around. When a
  **new schedule is installed** and we haven't accumulated real history
  yet, set `previous_week` in `themes.json` to a **later week** (from the
  same schedule) as a teaser — gives day-one users two weeks of content
  to browse. Remove the `previous_week` override once the natural
  minus-one lookback has a real week to point at.

## Prompt fields

Each prompt has:

- `id` — short unique slug, e.g. `fno-1`. Never renumber; past generations reference them.
- `label` — tile label. 2-3 words.
- `prompt` — sent to flux-fill-pro at inpainting time. Describes the outfit + context lighting that matches the event. This runs on the user's photo.
- `sample_prompt` — sent to flux-schnell at sample-generation time. Always catalog-editorial, white background, invisible mannequin. This produces the tile thumbnail.
- `negative_prompt` — standard negatives: `blurry, distorted, extra limbs, deformed hands, bad anatomy, low quality, cartoon, painting, text, watermark`.

## Workflow

### To change a prompt on an existing week
1. Edit the outfit description here.
2. Tell Claude to sync → `src/lib/themes.json` updated.
3. Re-run sample generation for that week if the change is visual.

### To add a new week beyond week 16
Append another `## Week N — …` section and a JSON block. Use a new Monday
`week_of` date (continue the 7-day cadence). Pick a new slug. Generate
samples.

### Sample regeneration
```bash
source .env.local
node scripts/generate-samples.mjs <slug> --size 1024
```
(Cost: ~$0.012 per week at 512px, ~$0.05 per week at 1024px with flux-schnell.)
